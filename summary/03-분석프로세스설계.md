# 3장. 분석 프로세스의 설계

이 장에서는 RapidMiner Studio에서 분석 프로세스를 처음부터 설계하고 실행하는 방법을 단계별로 설명합니다.

---

## 3.1 새 프로세스 만들기

새 프로세스를 만드는 방법:
- Home Perspective에서 **"New Process"** 클릭
- 또는 **File > New Process** 메뉴 사용

새 프로세스를 만들면 자동으로 Design Perspective로 전환됩니다.

---

## 3.2 리포지토리 작업

### 예제 데이터 사용하기

RapidMiner Studio는 **"Samples"** 리포지토리에 다양한 예제 데이터를 제공합니다. 이를 통해 학습에 활용할 수 있습니다.

### 데이터를 프로세스에 로드하기

1. Repositories View에서 데이터 항목을 **Process View로 드래그 앤 드롭**
2. 자동으로 **"Retrieve"** 오퍼레이터가 생성됨
3. 또는 Operators View에서 "Retrieve" 오퍼레이터를 수동으로 추가하고 파라미터로 데이터 위치 지정

---

## 3.3 첫 번째 분석 프로세스

### 프로세스 구축 단계

#### 1단계: 데이터 소스 추가

Repositories View에서 데이터를 Process View로 드래그합니다. "Retrieve" 오퍼레이터가 자동 생성됩니다.

#### 2단계: 데이터 변환 오퍼레이터 추가

Operators View에서 필요한 오퍼레이터를 검색하여 Process View로 드래그합니다.

**오퍼레이터 추가 방법**:
- Operators View에서 **드래그 앤 드롭**
- Operators View에서 **더블클릭** (현재 프로세스에 추가)
- Process View의 빈 공간에서 **마우스 오른쪽 클릭 > Insert Operator**

#### 3단계: 오퍼레이터 연결

출력 포트에서 다음 오퍼레이터의 입력 포트로 **선을 드래그**하여 연결합니다.

- 연결선은 데이터 흐름을 나타냄
- 호환되는 포트끼리만 연결 가능 (색상으로 표시)
- 연결 삭제: 연결선 위에서 마우스 오른쪽 클릭 > Delete

#### 4단계: 파라미터 설정

오퍼레이터를 클릭하면 Parameters View에서 해당 오퍼레이터의 파라미터를 설정할 수 있습니다.

- **필수 파라미터**: 굵은 글씨로 표시
- 파라미터 값 변경 시 오퍼레이터 상태 표시등이 다시 노란색으로 변경

#### 5단계: 결과 포트 연결

마지막 오퍼레이터의 출력 포트를 **프로세스의 결과 포트**(Process View 오른쪽 가장자리)에 연결합니다. 이렇게 해야 프로세스 실행 후 결과가 Results Perspective에 표시됩니다.

### 프로세스 검증

프로세스가 올바르게 구성되면:
- 모든 오퍼레이터의 상태 표시등이 **노란색** (준비 완료)
- Problems View에 **항목 없음**

프로세스에 문제가 있으면:
- 오퍼레이터 상태 표시등이 **빨간색**
- Problems View에 오류/경고 메시지 표시
- **Quick Fix** 사용 가능 (자동 수정 제안)

---

## 3.4 프로세스 실행

### 실행 방법

| 방법 | 설명 |
|------|------|
| 재생 버튼 | 툴바의 큰 재생 버튼 클릭 |
| 메뉴 | Process > Run |
| 단축키 | **F11** |

### 실행 중 상태 확인

- 현재 실행 중인 오퍼레이터: 상태 표시등에 **녹색 재생 아이콘**
- 성공 실행 후: 상태 표시등이 **녹색으로 유지**
- 파라미터 변경 시: 해당 오퍼레이터 이후 모두 다시 **노란색**

### 일시 정지와 중단

| 동작 | 버튼 | 효과 |
|------|------|------|
| **일시 정지** | 일시 정지 버튼 | 현재 오퍼레이터 완료 후 멈춤, 중간 결과 확인 가능 |
| **계속** | 재생 버튼 (녹색) | 일시 정지된 프로세스 재개 |
| **중단** | 중지 버튼 | 현재 오퍼레이터 완료 후 완전히 중단 |

> **참고**: 중단 시 현재 실행 중인 오퍼레이터는 항상 완료됩니다. 매우 복잡한 오퍼레이터의 경우 완료에 오랜 시간이 걸릴 수 있습니다.

### 3.4.1 결과 확인

프로세스가 종료되면 RapidMiner Studio는 자동으로 **Results Perspective**로 전환됩니다.

- 결과가 표시되지 않으면: 마지막 오퍼레이터의 출력 포트가 **결과 포트에 연결되지 않았을** 가능성
- Problems View 확인하여 오류 확인

> **팁**: **F8** = Design Perspective, **F9** = Result Perspective로 빠르게 전환

### 3.4.2 브레이크포인트

**브레이크포인트(Breakpoint)**는 프로세스 실행을 특정 지점에서 멈추고 중간 결과를 확인하는 기능입니다.

#### 브레이크포인트 설정

- 오퍼레이터 **컨텍스트 메뉴** > "Breakpoint Before" 또는 "Breakpoint After"
- 단축키: **F7** (선택된 오퍼레이터 뒤에 브레이크포인트 추가/제거)

#### 동작 방식

- **Breakpoint After**: 오퍼레이터 실행 후 중단, 출력 포트 결과 표시
- **Breakpoint Before**: 오퍼레이터 실행 전 중단, 입력 포트 결과 표시
- 브레이크포인트가 있는 오퍼레이터: 하단 가장자리에 **빨간색 기호** 표시

#### 브레이크포인트 상태 확인

- 하단 왼쪽 상태 표시등: **빨간색** (프로세스 실행 중이나 일시 정지 상태)
- 재생 기호: **파란색 → 녹색**으로 변경
- 녹색 재생 버튼을 누르면 다음 브레이크포인트까지 계속 실행

#### 브레이크포인트의 장점

프로세스 설계를 변경하지 않고도 **중간 결과를 자유롭게 확인**할 수 있습니다. 이는 특히 복잡한 서브프로세스 내부의 중간 결과를 검사할 때 유용합니다.
