# 1장. 기본 용어

이 장에서는 데이터 마이닝 활용에 대한 동기를 부여하고, 가장 중요한 용어들을 소개합니다. 숙련된 데이터 마이닝 전문가이든 아니든, 이 장은 본 서적과 RapidMiner에서 사용되는 용어들을 이해하는 데 필수적입니다.

---

## 1.1 우연인가, 아닌가?

### 데이터 마이닝의 동기

간단한 사고 실험을 해봅시다. 유리잔이 높은 곳에서 떨어지면 깨진다는 것을 우리는 경험적으로 알고 있습니다. 반면 동전 던지기의 결과는 예측할 수 없습니다. 이 차이는 **우연과 필연의 차이가 아니라**, 우리가 관련 영향 요인을 인식하고 측정할 수 있는지의 차이입니다.

> **핵심 명제**: 우리가 인지할 수 있는 환경에서 일어나는 대다수의 과정은 우연의 결과가 아닙니다. 과정을 정확하게 기술할 수 없는 이유는 필요한 영향 요인을 인식하거나 측정할 수 없기 때문입니다.

### 인간의 유추 추론

떨어지는 유리잔의 결과를 예측할 때 우리 머릿속에서는 다음과 같은 일이 일어납니다:

1. **데이터 수집**: 사건의 특성(재질, 높이, 바닥 등)을 측정
2. **유사성 비교**: 과거 유사한 경험과 비교
3. **예측 생성**: 가장 유사한 사례들의 결과를 바탕으로 추정

이 과정은 본질적으로 **데이터 마이닝의 원리**와 동일합니다.

### 경험 법칙의 생성

"1.5미터 이상 높이에서 떨어지는 유리잔은 깨진다"와 같은 **규칙(Rule)**을 도출할 수 있습니다. 이러한 임계값 기반 규칙은 항상 정확하지는 않지만, 대부분의 경우 올바른 추정을 가능하게 합니다.

**중요한 통찰**: 물리 법칙(인과관계)을 모르더라도, 불완전한 데이터만으로도 충분히 유용한 예측이 가능합니다. 이것이 바로 데이터 마이닝의 힘입니다.

## 1.2 기본 용어

### 1.2.1 속성과 목표 속성

**속성(Attribute)**은 시나리오를 설명하는 요인을 지칭하는 용어입니다. RapidMiner에서도 동일하게 사용됩니다.

| 동의어 | 영문 |
|--------|------|
| 특성 | Characteristic |
| 특징 | Feature |
| 영향 요인 | Factor |
| 지표 | Indicator |
| 변수 | Variable |
| 신호 | Signal |

**고객 시나리오 예시**: 기업이 새 제품에 관심을 가질 고객을 예측하려 할 때, 다음과 같은 속성을 사용합니다:
- 이름, 주소, 업종, 세부 업종, 직원 수, 제품 그룹별 구매 건수

이 중 **"프로토타입 긍정적 반응 여부"**는 특별한 속성입니다. 이것이 바로 **목표 속성(Target Attribute)** 또는 **레이블(Label)**입니다. 우리가 예측하고자 하는 대상입니다.

### 1.2.2 개념과 예제

- **개념(Concept)**: 속성으로 정의되는 유형 (예: "고객")
- **예제(Example)**: 개념의 구체적인 인스턴스 (예: "Doe Systems, Inc." - 기계업종, 직원 1000명 이상)
- **양성 예제**: 목표 속성 값이 "예"인 예제
- **음성 예제**: 목표 속성 값이 "아니오"인 예제

### 1.2.3 속성 역할

속성은 다양한 **역할(Role)**을 가질 수 있습니다:

| 역할 | 설명 |
|------|------|
| **Label** | 목표 속성 - 예측하고자 하는 값 |
| **ID (Identifier)** | 예제를 고유하게 식별하는 속성 |
| **Weight** | 레이블과 관련된 예제의 가중치 |
| **Regular** | 특별한 역할 없이 예제를 기술하는 일반 속성 |

### 1.2.4 값 유형

| 값 유형 | RapidMiner 명칭 | 용도 |
|---------|----------------|------|
| 명목형 | nominal | 범주형 비수치 값 (유한한 수의 카테고리) |
| 수치 값 | numeric | 일반적인 수치 값 |
| 정수 | integer | 양수 및 음수의 정수 |
| 실수 | real | 양수 및 음수의 실수 |
| 텍스트 | text | 구조가 없는 임의의 자유 텍스트 |
| 2값 명목형 | binominal | 두 개의 값만 허용 (예/아니오) |
| 다값 명목형 | polynominal | 두 개 이상의 서로 다른 값 |
| 날짜 시간 | date time | 날짜와 시간 모두 |
| 날짜 | date | 날짜만 |
| 시간 | time | 시간만 |

### 1.2.5 데이터와 메타 데이터

> **핵심 정의**:
> - **데이터(Data)** = 개념의 객체를 기술하는 구체적인 값들의 테이블
> - **메타 데이터(Meta Data)** = 개념(그리고 데이터)의 특성을 기술하는 정보

예제 집합(Example Set)은 속성이 열(Column), 예제가 행(Row)인 **테이블** 형태로 표현됩니다. RapidMiner에서 데이터(Data), 데이터 집합(Data Set), 예제 집합(Example Set)은 동의어로 사용됩니다.

> **실무 참고**: 데이터 분석 시간의 대부분은 데이터를 적합한 형식으로 변환하는 데 소요됩니다.

### 1.2.6 모델링

데이터가 구조화된 형식으로 준비되면, 데이터 마이닝 방법을 사용하여 **모델(Model)**을 생성합니다.

**모델의 예**:
> "고객이 도시 지역 출신이고, 500명 이상의 직원을 보유하며, 제품 그룹 1에서 최소 3건의 구매가 이루어졌다면, 이 고객이 새 제품에 관심을 가질 확률이 높다."

이러한 모델은:
- **이해 가능**: 기저 프로세스에 대한 통찰 제공
- **운영 가능(Operational)**: 새 고객에 대한 예측에 직접 사용 가능
- **효율적**: 무차별 마케팅 대신 유망 고객에 집중 가능

### RapidMiner Studio 소개

RapidMiner Studio는 기술과 사용성을 결합한 데이터 마이닝 플랫폼입니다:

- **1,500개 이상의 오퍼레이터** 제공
- **드래그 앤 드롭** 방식의 프로세스 설계
- **XML 기반** 내부 프로세스 구조
- **메타 데이터 변환**을 통한 설계 시점 검증
- **중단점(Breakpoint)** 지원으로 중간 결과 검사
- **빌딩 블록** 재사용 가능
- 텍스트 마이닝, 웹 마이닝, 감성 분석, 시계열 분석 지원
- 3D 그래프, 산점도 행렬, 자기 조직화 맵 등 강력한 시각화
