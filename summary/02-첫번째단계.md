# 2장. 첫 번째 단계

RapidMiner Studio의 설치부터 기본 인터페이스 사용법까지 안내합니다.

---

## 2.1 설치 및 첫 번째 리포지토리

### 설치

1. [http://www.rapidminer.com](http://www.rapidminer.com)에서 다운로드
2. 지원 OS: Windows, Macintosh, Linux/Unix
3. Linux/Unix의 경우 Java Runtime 최소 버전 7 필요

### 첫 번째 리포지토리 생성

RapidMiner Studio를 처음 시작하면 **새 리포지토리를 생성**하라는 메시지가 표시됩니다.

- **로컬 리포지토리**로 시작하는 것이 권장됨
- 이름(별칭)을 지정하고 하드 드라이브의 디렉토리를 선택
- 새 디렉토리를 만들어 로컬 리포지토리의 기반으로 사용하는 것이 좋음
- 리포지토리는 **데이터 및 분석 프로세스의 중앙 저장소** 역할

## 2.2 퍼스펙티브와 뷰

### 퍼스펙티브(Perspective)

RapidMiner Studio는 4개의 사전 정의된 퍼스펙티브를 제공합니다:

| 퍼스펙티브 | 설명 |
|-----------|------|
| **Home** | 프로그램 시작 후 환영 화면, 일반적인 작업 목록 |
| **Design** | 분석 프로세스를 생성, 편집, 관리하는 **중앙 작업 환경** |
| **Result** | 프로세스 결과(데이터, 모델 등)를 시각화 |
| **Wizard** | Application Wizard로 일반적인 데이터 마이닝 문제 해결 |

### Home Perspective의 주요 기능

1. **New Process**: 새 분석 프로세스 생성
2. **Open**: 기존 프로세스 열기 (최근 프로세스 목록 지원)
3. **Application Wizard**: 3단계로 데이터 마이닝 문제 해결
   - Direct Marketing Wizard: 최고 전환율 마케팅 활동 발견
   - Predictive Maintenance Wizard: 유지보수 활동 예측
   - Churn Analysis Wizard: 이탈 가능 고객 파악
   - Sentiment Analysis Wizard: 소셜 미디어 감성 분석
4. **Tutorials**: 내장 튜토리얼 제공

### 뷰(View) 관리

**뷰**는 퍼스펙티브를 구성하는 개별 UI 요소입니다.

- **추가**: View > Show View 메뉴에서 뷰 활성화
- **크기 변경**: 뷰 사이의 경계를 드래그
- **위치 변경**: 뷰의 이름 영역을 드래그하여 재배치
- **뷰 작업**: Close, Maximize, Minimize, Detach
- **복구**: View > Restore Default Perspective로 원래 상태 복구
- **사용자 정의**: New Perspective로 자신만의 퍼스펙티브 저장 가능

## 2.3 Design Perspective

Design Perspective는 RapidMiner Studio의 **핵심 작업 환경**입니다.

### 2.3.1 Operators 및 Repositories View

#### Operators View

RapidMiner Studio의 모든 오퍼레이터가 그룹별로 제공됩니다.

**주요 오퍼레이터 그룹**:

| 그룹 | 설명 |
|------|------|
| **Process Control** | 루프, 조건 분기 등 프로세스 흐름 제어 |
| **Utility** | 서브프로세스 그룹화, 매크로, 로깅 |
| **Repository Access** | 리포지토리 읽기/쓰기 |
| **Import** | 외부 형식(파일, DB 등)에서 데이터 읽기 |
| **Export** | 외부 형식으로 데이터 쓰기 |
| **Data Transformation** | 데이터/메타 데이터 변환 (가장 중요한 그룹) |
| **Modeling** | 분류, 회귀, 클러스터링, 연관 규칙 등 데이터 마이닝 |
| **Evaluation** | 모델 품질 평가 (교차 검증, 부트스트래핑 등) |

**오퍼레이터 검색 기능**:
- 이름 검색: 필터 필드에 입력
- **카멜 케이스 검색**: "REx" → "Read Excel", "DN" → "Date to Nominal/Numerical"
- 자동 연결: 드래그 앤 드롭 시 자동 연결 옵션 설정 가능

#### Repositories View

리포지토리는 RapidMiner Studio의 핵심 구성 요소로, 분석 프로세스를 프로젝트로 관리하고 구조화합니다.

> **경고**: RapidMiner 리포지토리를 사용할 것을 강력히 권장합니다. 그렇지 않으면 메타 데이터를 사용할 수 없게 되어 많은 지원 기능이 비활성화됩니다.

### 2.3.2 Process View

Process View는 분석 프로세스 내의 **개별 단계(오퍼레이터)와 그 연결**을 보여줍니다.

### 2.3.3 오퍼레이터와 프로세스

**오퍼레이터**는 다음으로 정의됩니다:
- 예상 **입력**에 대한 설명
- 제공되는 **출력**에 대한 설명
- 입력에 대해 수행하는 **동작**
- 동작을 제어하는 **파라미터**

**포트 시스템**:
- **입력 포트**: 오퍼레이터 왼쪽 (데이터 수신)
- **출력 포트**: 오퍼레이터 오른쪽 (결과 전달)
- 포트 색상으로 입력 유형 표시 (예: 푸른색 = 예제 세트)
- 빨간색 = 문제 있음, 흰색 = 결과 불분명

**오퍼레이터 유형**:
- 데이터 소스: 리포지토리/DB/파일에서 데이터 가져오기 (입력 포트 없음)
- 데이터 변환: 입력을 변환하여 동일 유형 반환
- 모델 생성: 입력을 소비하고 완전히 새로운 객체(모델) 생성

**상태 표시등**:
- 빨간색: 필수 조건 미충족 (오류)
- 노란색: 준비 완료 (경고 또는 정상)
- 녹색: 성공적으로 실행됨

### 2.3.4~2.3.8 기타 뷰 요약

| 뷰 | 기능 |
|----|------|
| **Parameters View** | 선택된 오퍼레이터의 파라미터 설정 |
| **Help View** | 선택된 오퍼레이터의 문서 표시 |
| **Problems View** | 프로세스의 오류/경고 목록 (Quick Fix 제공) |

**메타 데이터 변환**: 설계 시점에 기저 메타 데이터를 변환하여 결과 형태를 예견하고, 부적합한 오퍼레이터 조합의 해결책을 제안합니다.

**프로세스 자동 구성**: Quick Fix를 통해 RapidMiner Studio가 자동으로 문제를 해결하거나 제안합니다.
