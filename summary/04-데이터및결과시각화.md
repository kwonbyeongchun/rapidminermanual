# 4장. 데이터 및 결과 시각화

이 장에서는 RapidMiner Studio의 Results Perspective에서 데이터와 분석 결과를 확인하고 시각화하는 방법을 설명합니다.

---

## 4.1 결과 시각화

프로세스 완료 후, 결과 포트에 배치된 객체는 **Results Perspective에 자동으로 표시**됩니다.

- 각 결과는 **별도 탭**으로 표시
- 탭을 드래그하여 여러 결과를 **동시에 확인** 가능
- 새 결과 표시 전 이전 결과는 자동 닫힘

### 4.1.1 결과 표시를 위한 소스

결과를 표시하는 **3가지 방법**:

#### 1. 자동 열기 (권장)

프로세스의 **결과 포트에 연결된 출력**이 자동으로 표시됩니다. 브레이크포인트의 경우에도 동일합니다.

> 보고 싶은 모든 결과를 결과 포트에 연결하면, Results Perspective 탭에서 모두 확인 가능합니다.

#### 2. 리포지토리에서 열기

리포지토리 항목을 **더블클릭**하거나 컨텍스트 메뉴를 통해 열 수 있습니다. 이전 결과와의 비교에 유용합니다.

#### 3. 포트에서 열기

실행된 오퍼레이터의 출력 포트에서 **컨텍스트 메뉴 > Show result** 선택.

> **주의**: 포트의 결과는 메모리 절약을 위해 일정 시간 후 **자동 삭제**될 수 있습니다. 따라서 결과 포트를 통한 자동 표시(방법 1)를 권장합니다.

## 4.2 데이터 복사본과 뷰에 대하여

RapidMiner Studio는 효율성을 위해 **불필요한 데이터 복사본을 생성하지 않습니다.** 이로 인해 발생할 수 있는 혼란과 해결 방법:

### 문제 상황

정규화 오퍼레이터가 기본 데이터를 **직접 변경**합니다. 정규화 이전 포트에서 데이터를 확인해도 이미 변경된 데이터가 보일 수 있습니다.

### 해결 방법

#### 방법 1: 뷰(View) 사용

많은 데이터 변환 오퍼레이터에 **"create view"** 파라미터가 있습니다.
- 데이터를 직접 변경하지 않고 **뷰를 추가**
- 데이터 접근 시 즉석에서(on-the-fly) 변경
- 이전 포트나 병렬 분기에 영향 없음

#### 방법 2: 명시적 복사

**Multiply** 오퍼레이터 + **Materialize Data** 오퍼레이터 조합:
1. Multiply로 데이터 참조를 복제
2. Materialize Data로 실제 데이터 복사본 생성

> 작은 데이터 세트에 적합하며, 병렬 분기에서 독립적인 데이터 처리가 필요한 경우 사용합니다.

## 4.3 표시 형식

각 결과는 다양한 **뷰(View)**로 표시될 수 있습니다.

데이터 세트의 경우:
- **Data View**: 데이터 자체의 테이블 표시
- **Statistics View**: 메타 데이터 및 통계
- **Charts View**: 다양한 시각화
- **Advanced Charts View**: 고급 시각화

### 4.3.1 설명 (Description)

**텍스트 형식**의 가장 기본적인 시각화입니다.
- 일부 모델과 결과를 텍스트로 표시
- **Ctrl+A** → **Ctrl+C**로 텍스트 복사 가능

### 4.3.2 테이블 (Tables)

RapidMiner에서 가장 빈번한 정보 표시 형식입니다.

#### 색상 코딩

| 요소 | 색상 |
|------|------|
| 특별한 역할 속성 (Label, ID 등) | 연한 노란색 배경 |
| 일반 속성 | 연한 파란색/흰색 교대 |
| 상관관계 행렬 | 색상 진할수록 강한 상관관계 |

#### 정렬

- 열 머리글 **클릭**: 오름차순/내림차순 정렬 (삼각형 표시)
- **세 번째 클릭**: 정렬 비활성화
- **Ctrl + 클릭**: 다중 열 정렬 (최대 3개 열)

> **참고**: 큰 테이블에서는 성능을 위해 정렬이 비활성화될 수 있습니다. 임계값은 Tools > Preferences에서 설정합니다.

#### 열 조작

- **열 이동**: 열 머리글 드래그
- **열 너비 조정**: 열 경계 드래그 또는 더블클릭 (자동 조정)
- **모든 열 자동 조정**: **Ctrl + 열 경계 더블클릭**

#### 컨텍스트 메뉴 주요 동작

| 동작 | 설명 |
|------|------|
| Select Row/Column | 행/열 선택 |
| Fit Column Width | 선택 열 너비 조정 |
| Fit all Column Widths | 모든 열 너비 조정 |
| Sort by Column | 열 기준 정렬 |
| Sort Columns by Names | 열 이름 알파벳순 재배치 |
| Restore Column Order | 원래 열 순서 복원 |

#### 복사

- 개별 셀 선택 또는 **Ctrl+A**로 전체 선택
- **Ctrl+C**로 클립보드에 복사
- Excel 등에 붙여넣기 시 테이블 구조 유지

### 4.3.3 차트 (Charts)

Charts View와 Advanced Charts View에서 **30가지 이상의 시각화 방법**을 제공합니다.

#### 차트 구성

- **왼쪽 구성 영역**에서 시각화 유형 선택 (가장 중요한 설정)
- 차트 유형에 따라 설정 필드가 동적으로 변경

#### 주요 차트 유형

| 유형 | 설명 |
|------|------|
| **Scatter** | 산점도 - x축, y축 속성 지정, 색상으로 세 번째 속성 표현 |
| **Bars / Bars Stacked** | 막대 그래프 - 그룹화/스택 속성, 집계 함수 설정 |
| **Lines** | 선 그래프 |
| **Pie** | 파이 차트 |
| **3D** | 3차원 시각화 |
| **SOM** | 자기 조직화 맵 (Calculate 버튼으로 계산 필요) |

#### 유용한 팁

- **Jitter 기능**: 명목 값을 포함한 데이터에서 겹치는 점을 분산 표시
- **로그 스케일**: 축을 로그 스케일로 조정 가능
- **텍스트 회전**: x축의 긴 텍스트를 회전시켜 가독성 향상
- **색상 설정**: Tools > Preferences에서 변경 가능

### 4.3.4 그래프 (Graphs)

노드와 그 관계를 보여주는 시각화입니다.

**대표적 예시**: 의사결정 트리, 계층적 클러스터링

#### 탐색 모드

| 모드 | 설명 |
|------|------|
| **이동 모드** | 마우스 드래그로 그래프 영역 이동 |
| **선택 모드** | 개별 노드 클릭 선택, 드래그로 다중 선택, Shift+클릭으로 추가/제외 |

- **확대/축소**: 마우스 휠 또는 구성 영역의 +/- 버튼
- **레이아웃 선택**: 다양한 레이아웃 알고리즘 중 선택 가능
- **캡션 표시**: 노드/에지 캡션 표시 여부 설정 가능

### 4.3.5 특수 뷰

설명, 테이블, 차트, 그래프 외에도 특수한 표시 구성 요소가 있습니다.
- 예: 빈발 항목 집합(Frequent Itemsets)의 연관 규칙 테이블/그래프

## 4.4 결과 개요 (Result Overview)

Result Overview는 현재 RapidMiner 세션의 **모든 프로세스 실행 결과에 대한 간략한 개요**입니다.

### 표시 정보

각 항목에 표시되는 정보:
- 프로세스 이름
- 결과 수
- 완료 시점
- 실행 시간

### 사용법

- 항목 **클릭**: 결과 상세 보기 (예: 예시 세트, 의사결정 트리)
- **다시 클릭**: 닫기
- **여러 항목 동시 열기**: 결과 비교에 유용
- 동일 프로세스의 결과 블록은 **교대 색상**으로 구분

### 항목별 동작

| 동작 | 설명 |
|------|------|
| 프로세스 복원 | 해당 항목의 프로세스를 원래 형태로 복원 |
| 항목 삭제 | Result Overview에서 해당 항목 제거 |
| 전체 삭제 | 컨텍스트 메뉴에서 전체 개요 삭제 |
